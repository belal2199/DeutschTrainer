## FEATURE 1 — Inline Sentence Corrector (per vocabulary card)

### Where it appears
Every vocabulary card in the Verben, Nomen, and Adjektive views has an existing "example sentence" display. Below that example sentence, add a small expandable practice area.

### UI behaviour
1. A small button labelled "Satz üben" appears below the example sentence on each vocabulary card.
2. Clicking it expands an inline practice panel — it does NOT open a new page.
3. The panel contains:
   - A label: "Schreib einen Satz mit '[word]':"
   - A single-line text input (placeholder: "Dein Satz auf Deutsch…")
   - A button: "Korrigieren"
   - A result area below (hidden until API responds)
4. When the user clicks "Korrigieren":
   - Show a subtle loading spinner inside the button
   - Call the backend correction API (see below)
   - Display the result in the result area
5. The result area shows three things:
   - A status badge: "Korrekt" (green) or "Fast korrekt" (amber) or "Mit Fehlern" (red)
   - The corrected sentence in a styled box (use a different colour from the input if corrections were made; show original if already correct)
   - A short explanation in simple German (B1–B2 level) of what was wrong, if anything. If nothing is wrong, show a short encouraging message.
6. The panel stays open until the user clicks "Schließen" or clicks "Satz üben" again to toggle it closed.
7. Each card manages its own panel state independently — opening one does not close others.

## BACKEND — Correction API

### New endpoint
POST /api/correct

Request body:
{
  "text": "string",           // the user's German sentence or text
  "context"?: "string",       // optional: the vocabulary word being practised (Feature 1 only)
  "mode": "sentence" | "free" // "sentence" for Feature 1, "free" for Feature 2
}

Response body:
{
  "original": "string",
  "corrected": "string",
  "isCorrect": boolean,
  "status": "correct" | "minor" | "errors",
  "corrections": [
    {
      "original": "string",      // wrong phrase
      "corrected": "string",     // correct phrase
      "explanation": "string",   // simple German explanation (B1-B2)
      "type": "grammar" | "spelling" | "word_order" | "vocabulary" | "article"
    }
  ],
  "overallFeedback": "string",   // short encouraging German sentence
  "processingTime": number       // milliseconds
}

### AI prompt (use this exact system prompt for the language model)

SYSTEM PROMPT:
"""
Du bist ein freundlicher Deutschlehrer für B1–B2 Lernende. Deine Aufgabe ist es, deutsche Sätze und Texte zu korrigieren.

Regeln:
1. Korrigiere NUR echte Fehler (Grammatik, Rechtschreibung, Wortstellung, falsche Artikel, falsches Verb).
2. Verändere NICHT den Stil oder das Vokabular des Lernenden — auch wenn du es anders formulieren würdest.
3. Erkläre jeden Fehler in einfachem Deutsch (B1–B2 Niveau) — kurz und verständlich.
4. Sei immer ermutigend und positiv.
5. Wenn kein Fehler vorliegt, lobe den Lernenden kurz.
6. Antworte IMMER als valides JSON-Objekt mit dieser exakten Struktur:
{
  "corrected": "string",
  "isCorrect": boolean,
  "status": "correct" | "minor" | "errors",
  "corrections": [
    {
      "original": "string",
      "corrected": "string",
      "explanation": "string",
      "type": "grammar" | "spelling" | "word_order" | "vocabulary" | "article"
    }
  ],
  "overallFeedback": "string"
}

Kein Markdown, keine Erklärungen außerhalb des JSON.
"""

USER PROMPT for sentence mode:
"""
Bitte korrigiere diesen deutschen Satz. [Wenn Kontext vorhanden: Der Lernende übt das Wort "[context]".]

Satz: "[text]"
"""

USER PROMPT for free mode:
"""
Bitte korrigiere diesen deutschen Text. Finde alle Fehler und erkläre sie.

Text: "[text]"
"""

### Environment variable
Add to .env:
  OPENAI_API_KEY=your_key_here
  AI_MODEL=gpt-4o              # configurable, default gpt-4o
  AI_MAX_TOKENS=1000
  AI_TEMPERATURE=0.2           # low temperature for consistent corrections

### Error handling
- If the AI API is unreachable or returns an error: respond with HTTP 503 and message "KI-Service nicht verfügbar. Bitte versuche es später."
- If the text is empty: respond with HTTP 400 and message "Bitte gib einen Text ein."
- If the text is longer than 2000 characters: respond with HTTP 400 and message "Text ist zu lang. Maximal 2000 Zeichen."
- If the AI response is not valid JSON: retry once, then return 500.
- Add basic rate limiting: max 20 requests per minute per IP.

## FRONTEND — Angular service

### New service: CorrectionService (correction.service.ts)
Methods:
  correctSentence(text: string, context?: string): Observable<CorrectionResult>
  correctFreeText(text: string): Observable<CorrectionResult>

The service stores the last 5 correction results in a signal:
  correctionHistory = signal<CorrectionHistoryItem[]>([])

### New shared interface (shared/types/correction.types.ts)
export interface CorrectionResult {
  original: string;
  corrected: string;
  isCorrect: boolean;
  status: 'correct' | 'minor' | 'errors';
  corrections: CorrectionItem[];
  overallFeedback: string;
  processingTime: number;
}

export interface CorrectionItem {
  original: string;
  corrected: string;
  explanation: string;
  type: 'grammar' | 'spelling' | 'word_order' | 'vocabulary' | 'article';
}

export interface CorrectionHistoryItem {
  timestamp: Date;
  preview: string;
  status: 'correct' | 'minor' | 'errors';
  result: CorrectionResult;
}

### Correction type badge colours
  correct   → green  (same as --accent / #4ade80)
  minor     → amber  (#f59e0b)
  errors    → red    (#f87171)
